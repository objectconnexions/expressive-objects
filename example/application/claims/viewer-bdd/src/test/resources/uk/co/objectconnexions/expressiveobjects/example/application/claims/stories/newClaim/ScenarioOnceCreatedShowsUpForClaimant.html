New Claim once created shows up for claimant

Given

Given this fixture:

Locate and alias Tom:

  • With the employees service, invoke action All Employees and alias the resulting list as list1; ok

  • Alias Tom Brown in list1 as tomEmployee; ok.

Initially Tom has no claims:

  • Using tomEmployee, invoke action Claims For and alias the resulting list as tomsClaimsInitially; ok.

  • Confirm that tomsClaimsInitially has no claims; ok.

When

Create new claims, and save:

on object alias result as perform on member value
tomEmployee tomsClaim1 invoke action New Claim ok
tomsClaim1 set property Description claim 1 ok
tomsClaim1 set property Date 14-02-2007 ok
tomsClaim1 save ok
tomEmployee tomsClaim2 invoke action New Claim ok
tomsClaim2 set property Description claim 2 ok
tomsClaim2 set property Date 18-02-2007 ok
tomsClaim2 save ok

run viewer

Then

Tom now has claims:

  • Using tomEmployee, invoke action Claims For and alias the resulting list as tomsClaimsAfterwards; ok.

  • Confirm that the tom's claims afterwards is not empty; ok.

  • Confirm that tom has 2 claims; ok.

  • Confirm that tom's set of claims contains tomsClaim1; ok.

  • Confirm that tom's set of claims does NOT contain tomEmployee; himself (duh!); ok.

  • Tom has these claims (checking using ExpressiveObjects' checkList):

    title
    New - 2007-02-18 ok
    New - 2007-02-14 ok

  • Tom has these claims (checking using Concordion's verifyRows):

    Description Date
    claim 2 2007-02-18
    claim 1 2007-02-14
