New Feature

  • [EXPRESSIVE_OBJECTS-109] - JSON viewer (for Restful Objects, up to v0.56).

  • [EXPRESSIVE_OBJECTS-124] - Add a @Stable annotation in the applib (along with StableFacet and StableAnnotationFacetFactory in the metamodel)
  • [EXPRESSIVE_OBJECTS-128] - Add DomainObjectContainer#titleOf(pojo) to allow domain objects to programmatically obtain the title of other domain objects.
  • [EXPRESSIVE_OBJECTS-130] - New "onlinedemo" example to demonstrate combined html and json viewers as live example.
  • [EXPRESSIVE_OBJECTS-135] - Allow HTML viewer to co-exist with other viewers, (by supporting a user-defined suffix for its links; eg ".htmlviewer" rather than ".app")
  • [EXPRESSIVE_OBJECTS-136] - Allow json viewer to coexist with other viewers (by introducing new authentication session lookup strategy to read from headers and enhancing ExpressiveObjectsSessionFilter)
  • [EXPRESSIVE_OBJECTS-137] - Provide the ability to allow new users to register through the HTML viewer.
  • [EXPRESSIVE_OBJECTS-142] - Provide "String disabled()" method on objects to disable the entire object at runtime.
  • [EXPRESSIVE_OBJECTS-143] - Provide "Boolean hidden()" method on objects to hide the entire object at runtime.
  • [EXPRESSIVE_OBJECTS-147] - Adding new features to the SQL objectstore
  • [EXPRESSIVE_OBJECTS-149] - Update ExpressiveObjectsTestRunner to accept annotations to override Persister and Authenticator
  • [EXPRESSIVE_OBJECTS-160] - Allow users to override "expressive-objects_" table prefix
